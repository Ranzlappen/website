<h2 id="overview">Overview</h2>

The Huawei Watch D2 is a medical-grade wearable best known for its cuff-based blood-pressure measurement<sup><a href="#source-2">[2]</a></sup>. The official watch faces cover the basics — time and health stats — but none of them matched the clean, dark, information-dense layout I wanted on my wrist. **D2App** is the result: a custom HarmonyOS application written in ArkTS that replaces the stock face with a minimal dark-themed watch face showing a live digital clock, real-time heart rate, and battery level, optimised for the Watch D2's round screen.

What made this project interesting beyond the UI work is the build pipeline. Writing, compiling, and sideloading HarmonyOS apps requires Huawei's DevEco Studio IDE and a chain of signing certificates, device UDID registration, and provisioning profiles through AppGallery Connect<sup><a href="#source-3">[3]</a></sup>. D2App automates the compilation step entirely via GitHub Actions: every push to `main` produces a signed `.hap` artifact that can be installed directly — no local Huawei toolchain required for regular updates once the signing setup is done.

<h2 id="features">Features</h2>

- **Live digital clock** — Displays the current time in a large, high-contrast typeface centred on the round screen. The layout is tuned for the Watch D2's circular display geometry — no corners wasted, no clipping.
- **Real-time heart rate monitoring** — Reads from the watch's optical heart rate sensor<sup><a href="#source-2">[2]</a></sup> and updates the displayed value continuously. The sensor subscription is managed via HarmonyOS's health sensor APIs in ArkTS.
- **Battery level indicator** — Shows the current battery percentage using the system battery API, positioned so it doesn't compete visually with the clock or heart rate readout.
- **Tappable interaction area** — A defined region of the face responds to tap events, triggering a state change or a secondary screen — a deliberate affordance that the stock faces don't expose in a customisable way.
- **Dark theme, round-screen optimised** — The entire UI uses near-black backgrounds with bright accent colours, designed specifically for the AMOLED circular panel so active pixels are minimised and contrast is maximised.
- **CI/CD-automated `.hap` builds** — GitHub Actions compiles the ArkTS source with Hvigor<sup><a href="#source-4">[4]</a></sup> on every push to `main`. The resulting `.hap` is available as a workflow artifact — no local DevEco Studio installation needed to pick up a new build.

<h2 id="how-it-works">How It Works</h2>

**Language and runtime — ArkTS on HarmonyOS Stage Model**

HarmonyOS applications use ArkTS<sup><a href="#source-5">[5]</a></sup> — a TypeScript superset that adds declarative UI syntax (ArkUI) and access to system APIs. The Stage Model is the current application lifecycle model for HarmonyOS 4+, replacing the older FA Model. It organises code into abilities (entry points) and pages (UI screens). D2App has a single entry ability and one main page, `entry/src/main/ets/pages/Index.ets`, which contains all the clock, sensor, and battery logic.

**Sensor and battery subscriptions**

ArkTS sensor APIs are subscription-based. The heart rate subscription looks roughly like:

```typescript
sensor.on(sensor.SensorId.HEART_RATE, (data: sensor.HeartRateResponse) => {
  this.heartRate = data.heartRate;
});
```

The clock updates via a `setInterval` equivalent (HarmonyOS timer API) rather than a sensor subscription since time is not a push-based sensor. Battery level uses `batteryInfo.batterySOC` from the `@ohos.batteryInfo` module, polled or event-driven depending on the HarmonyOS version.

**Build system — Hvigor**

Hvigor<sup><a href="#source-4">[4]</a></sup> is Huawei's build tool for HarmonyOS (analogous to Gradle on Android). The GitHub Actions workflow installs the required SDK components, runs `hvigorw assembleHap --mode module`, and uploads the produced `.hap` as an artifact. Because the build is self-contained in CI, contributors don't need DevEco Studio to iterate on layout changes — edit `Index.ets`, push, and download the artifact.

**Installation path**

Installing a self-signed `.hap` requires the device UDID to be registered in AppGallery Connect and a debug provisioning profile generated with that UDID included. Two installation methods work:

- **Phone method**: Pair the watch via Bluetooth to a phone running HUAWEI DevEco Assistant, then push the `.hap` from the Assistant app.
- **Computer method (HDC)**: Enable USB debugging on the watch and use the `hdc` command-line tool (`hdc install <app>.hap`) over a USB connection.

**Project structure**

```
D2app/
├── entry/
│   └── src/main/ets/pages/
│       └── Index.ets          # Clock, HR sensor, battery UI
├── .github/workflows/          # Automated .hap build

```

<h2 id="getting-started">Getting Started</h2>

**Install a pre-built artifact (easiest)**

1. Visit the [Actions tab](https://github.com/Ranzlappen/D2app/actions) of the repository, open the latest successful workflow run, and download the `.hap` artifact.
2. Register your Watch D2's UDID in [AppGallery Connect](https://developer.huawei.com/consumer/en/agconnect/)<sup><a href="#source-3">[3]</a></sup> under your developer account and regenerate the debug signing certificate to include it.
3. Install via the phone method (HUAWEI DevEco Assistant Bluetooth pairing) or the HDC computer method (USB debugging).

**Build from source**

Requirements: DevEco Studio 4.x with the HarmonyOS 4 SDK, a Huawei Developer account, and a registered device UDID.

```bash
git clone https://github.com/Ranzlappen/D2app
cd D2app
# Open in DevEco Studio and configure signing in File → Project Structure → Signing Configs
# Or via CLI:
./hvigorw assembleHap --mode module
```

**Customise the face**

All UI code is in `entry/src/main/ets/pages/Index.ets`. Clock styling, heart rate display format, tap handlers, and colour values are all in one file. Edit, commit to `main`, and the Actions workflow produces a fresh `.hap` automatically.

<h2 id="key-takeaways">Key Takeaways</h2>

- ArkTS's declarative UI model (ArkUI) is structurally similar to SwiftUI or Jetpack Compose — once you're familiar with reactive/declarative patterns, the concepts transfer even if the APIs are vendor-specific.
- Automating `.hap` builds in CI is straightforward once the Hvigor toolchain is scripted; the harder part is the one-time Huawei developer account and UDID registration ceremony.
- The Watch D2's round display requires explicit layout thought that rectangular-screen phone development doesn't — element sizing, radial positioning, and how text renders at small sizes near the screen edge all need deliberate attention.
- Publishing even a small wearable project as open source is valuable because HarmonyOS development documentation is sparse in English; a working example repository does more for discoverability than any number of forum posts.

<h2 id="conclusion">Conclusion</h2>

D2App is a small project by line count — a single ArkTS page driving three sensor streams — but it covers a development workflow that is surprisingly underdocumented outside of Huawei's own Chinese-language ecosystem. If you own a Huawei Watch D2 and want a minimal, open-source starting point for a custom watch face, the repository has everything you need: working ArkTS sensor subscriptions, a round-screen-optimised layout, and a GitHub Actions pipeline that builds a ready-to-install `.hap` on every commit.
