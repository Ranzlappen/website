<h2 id="overview">Overview</h2>

Every JPEG you shoot carries invisible baggage: GPS coordinates, camera serial number, the exact timestamp the shutter fired, colour-profile data, software version strings, and dozens of other tags baked in by your camera, phone, or editing app. Most people never see this data — until they post a photo publicly and realise they've just shared their home address with the internet.

**MetaTable** (the project behind this post, hosted as [Ranzlappen/Exif](https://github.com/Ranzlappen/Exif)) is a portable, single-executable Windows tool that makes that hidden layer visible, editable, and — when you want — gone. You drop a file onto the window, see every metadata tag in a sortable, searchable table, edit values inline, and write the changes back with an automatic backup created before anything is touched. No installation, no Python runtime on the target machine, no internet connection required.

The project runs on top of Phil Harvey's [ExifTool](https://exiftool.org/)<sup><a href="#source-2">[2]</a></sup> — the de-facto standard metadata engine supporting 400+ file formats — fronted by a GPU-accelerated [Dear PyGui](https://github.com/hoffstadt/DearPyGui)<sup><a href="#source-3">[3]</a></sup> interface and packaged into a self-contained `.exe` via [PyInstaller](https://pyinstaller.org/)<sup><a href="#source-4">[4]</a></sup>.

<h2 id="features">Features</h2>

- **Universal metadata coverage** — Reads EXIF, IPTC, XMP, QuickTime, ID3, PDF metadata, and Office document properties. Because ExifTool does the heavy lifting, "400+ formats" is not marketing copy: it covers JPEG, PNG, TIFF, CR2/CR3, NEF, ARW, DNG, HEIC, MP4, MOV, MKV, MP3, FLAC, WAV, PDF, DOCX, and many more<sup><a href="#source-2">[2]</a></sup>.
- **Interactive sortable/searchable table** — All tags display in a Dear PyGui table with columns for Group, Tag name, current Value, Writable status, and a human-readable Description. Click any column header to sort; type in the search box to filter instantly across thousands of tags.
- **Inline editing with safety backups** — Click any writable value, type a new one, confirm, and ExifTool writes the change — but only after creating a `<filename>_original` backup automatically. No data loss is possible from an accidental edit.
- **Batch file and folder loading** — Drop an entire folder onto the app to load all supported files at once. Switch between loaded files via a sidebar list without re-opening dialogs.
- **Custom tag insertion** — A searchable dialog lets you add tags that don't already exist in a file — useful for embedding copyright notices, licensing terms, or custom IPTC fields into images before distribution.
- **Portable single executable** — The compiled build (~15–20 MB) bundles the Python runtime, Dear PyGui, and ExifTool internally. Copy it to a USB drive and run it on any Windows machine without installing anything.
- **Dark theme throughout** — The interface uses Dear PyGui's dark theme with no light-mode variant, consistent with the tool's "utility first" focus.

<h2 id="how-it-works">How It Works</h2>

The architecture has three layers that fit together cleanly.

**Metadata engine — ExifTool via persistent subprocess**

Rather than spawning a new ExifTool process for each file (slow on large batches), MetaTable opens ExifTool once in `-stay_open` mode<sup><a href="#source-2">[2]</a></sup> — a flag that keeps the Perl process alive and accepting commands on stdin, with results returned on stdout terminated by a sentinel string. This drops per-file overhead from hundreds of milliseconds to single-digit milliseconds once the process is warm.

Reading a file sends `exiftool -json -G -a -u -D <path>` through the pipe. The `-G` flag groups tags by family (EXIF, IPTC, XMP, …), `-a` allows duplicate tag names across groups, and `-D` returns numeric tag IDs alongside human names — the combination gives enough context to identify exactly which tag to write back without ambiguity.

Writing calls `exiftool -<TagName>=<value> -overwrite_original_in_place <path>` (or the equivalent through the stay-open pipe), which triggers ExifTool's built-in backup mechanism before modifying the file.

**GUI — Dear PyGui**

Dear PyGui is an immediate-mode, GPU-accelerated Python GUI library built on Dear ImGui<sup><a href="#source-3">[3]</a></sup>. The table widget handles the full metadata list — often 300–500 rows per file — without virtualisation lag because ImGui renders only the visible rows each frame. Sorting and filtering are implemented as in-memory list operations on the parsed tag list; the table re-renders from the filtered subset on the next frame.

**Packaging — PyInstaller**

`build.bat` runs PyInstaller in `--onefile` mode, embedding the Python interpreter, all pip dependencies, and a bundled copy of ExifTool's executable into a single `.exe`. The resulting file is self-extracting at launch (to a temp folder) and cleans up on exit, so from the user's perspective it is always one file.

**Source structure**

```
Exif/
├── main.py           # Application entry point and GUI layout
├── requirements.txt  # Dear PyGui + PyInstaller
├── build.bat         # PyInstaller one-file build script
└── .gitignore
```

<h2 id="getting-started">Getting Started</h2>

**Option A — Run the portable executable (recommended)**

1. Go to the [Releases page](https://github.com/Ranzlappen/Exif/releases) and download the latest `MetaTable.exe`.
2. Double-click it. No installation wizard, no UAC prompts beyond the first run, no internet connection needed.
3. Drag any image, video, audio, or document file onto the window (or use File → Open) to load its metadata.

**Option B — Run from source**

Requirements: Python 3.11+<sup><a href="#source-5">[5]</a></sup> and [ExifTool](https://exiftool.org/) added to your `PATH`.

```bash
git clone https://github.com/Ranzlappen/Exif
cd Exif
pip install -r requirements.txt
python main.py
```

**Option C — Build your own executable**

```bash
# Prerequisites: Python 3.11+, requirements installed, ExifTool in PATH
build.bat
# Outputs dist/MetaTable.exe (~15–20 MB)
```

<h2 id="roadmap">Roadmap</h2>

- [ ] Multi-file batch editing — apply the same tag change (e.g. strip all GPS data, set copyright string) across an entire loaded folder in one operation
- [ ] GPS map preview — when a file contains GPS coordinates, show a static map pin in a sidebar panel
- [ ] CSV / JSON export — dump the full tag table to a spreadsheet-friendly format for archiving or auditing large photo libraries
- [ ] Custom presets — save a named set of tag values (e.g. "strip personal data", "add studio copyright") and apply them in one click
- [ ] Undo/redo — track edit history in-session so changes can be reversed without falling back to the `_original` backup
- [ ] Cross-platform packaging — macOS `.app` and Linux AppImage builds via PyInstaller on those platforms

<h2 id="key-takeaways">Key Takeaways</h2>

- ExifTool's `-stay_open` mode turns what would be a slow per-file process spawn into a near-instant IPC channel — the right tool primitive makes the whole architecture simpler and faster.
- Dear PyGui's immediate-mode model is a surprisingly good fit for data-heavy utility apps: no widget state to synchronise, no diffing, just "render what's in the list this frame."
- Bundling everything into a single `.exe` with PyInstaller is genuinely powerful for utility software aimed at non-developers — "one file, just run it" removes the biggest adoption barrier.
- Automatic backups before any write operation should be the default in every metadata editor; the `_original` convention ExifTool uses is a good pattern to follow in any destructive-edit tool.

<h2 id="conclusion">Conclusion</h2>

MetaTable fills a gap between heavy professional tools like Adobe Bridge and the command-line-only world of raw ExifTool usage. It gives anyone — photographer, archivist, or privacy-conscious sharer — a clear window into what their files are actually carrying, and the ability to clean house with one click.

The project is open source and portable enough to carry on a USB stick. Give it a try, and if you run into a format it doesn't handle well, ExifTool's support list is the benchmark: if ExifTool reads it, MetaTable should be able to show it.
